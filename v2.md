## 三、过程复盘与沉淀
### 1. 本周主要完成环节与工具方法
- **AI 与人的共生开发**：
  - **人负责顶层设计**：定义 E-commerce 和 New Media 的业务逻辑边界、定义 HTP 测评的临床交互路径。
  - **AI 负责细节实现**：生成复杂的 HTP 绘图逻辑、多表关联的状态管理代码、以及各模块的精美视觉样式。
- **跨端技术突破**：
  - 使用 **React Portals** 技术彻底解决了编辑器（EditableWrapper）在复杂嵌套 DOM 中因 Z-index 和 Overflow 导致的点击穿透与失效问题。

- **工具链集成**：
  - 深度使用 AI 的多轮交互能力重绘了 Assets 模块的视觉资产（如音乐盒封面、数字资产卡片）。

### 2. 遇到的问题与破局方法
- **问题 1：编辑器交互“死区”**
  - **现象**：在首页重构后，由于外层动画容器限制了 overflow 且 z-index 混乱，导致所有编辑按钮可见但点击无响应。
  - **破局**：弃用常规 CSS 堆叠，采用 `createPortal` 将交互层挂载到 body 顶层，彻底绕过父级 DOM 限制。
- **问题 2：多平台数据管理逻辑复杂**
  - **现象**：新媒体模块涉及 4 个平台，初始代码数据耦合严重，维护成本极高。
  - **破局**：抽象出平台无关的 `ContentLibrary` 和 `PublishingGrid` 通用逻辑，通过 Schema 驱动数据渲染，实现 10 分钟接入新平台。
- **问题 3：自动化脚本 PPT 布局错位**
  - **现象**：动态生成的图片与文字在 PPT 中经常重叠。
  - **破局**：引入硬编码布局规范与动态占位符检测机制，并在 `generate_ppt.py` 中增加了智能层叠逻辑。

### 3. 可复用的经验与资产
- **万能编辑包装器 (EditableWrapper Pattern)**：一种基于 Portal 的、无侵入式的 UI 编辑态装饰器模式。
- **AI 提示词套路**：“将以下业务逻辑逻辑转化为符合 Apple 审美风格的 React 组件，使用 Vanilla CSS，要求具有毛玻璃效果和细腻的微动效。”
- **业务组件库**：积累了高可用的订单管理列表、内容发布卡片、以及心理测试雷达图等业务模板。

## 四、本周随手记
- **PDF 打印心得**：在实现简历打印预览时，发现 `aspect-ratio` 属性可能导致打印分页裁剪，强制指定 `height: auto` 配合 `break-inside: avoid` 是最佳实践。
- **视觉美学**：使用 `mask-image` 和 `backdrop-filter` 配合可以做出极具质感的现代 UI，但要注意移动端 Safari 的前缀兼容。
- **效率建议**：当一个 UI 交互修了两次 CSS 还没好时，果断考虑用 Portal 或重构 DOM 结构，不要在父级的 `overflow-hidden` 里浪费时间。
- **AI 协作**：先给 AI 发一段项目的 `types.ts` 定义，再让它写逻辑，生成的代码精准度会提升一个数量级。
