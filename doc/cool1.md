# 🚀 Opclaw — 全能超级个体数字宇宙 · 项目全景文档

> **一个人的数字宇宙，从这里开始。**
> 个人 IP 展示 × 知识资产管理 × AI 数字分身 × 生活情感珍藏 × 工作效率加速

---

## 🌟 一、产品概述

### 1. 项目背景

| 维度 | 说明 |
|------|------|
| 🌍 **时代趋势** | 个体经济崛起，中国自由职业者超 **2 亿**，"超级个体"概念爆发 |
| 🧩 **行业痛点** | 个人品牌工具割裂——简历在 Word、作品在 GitHub、社交在各平台、笔记在 Notion |
| 🤖 **AI 浪潮** | 大模型普及，但通用 AI 不了解用户个人知识体系，缺乏个性化 |
| 📱 **市场空白** | 市面无产品将"展示 + 知识 + AI + 生活 + 工作"五合一融合 |

### 2. 项目简介

| 项目 | 内容 |
|------|------|
| 📛 **产品名称** | Opclaw（OPC 超级个体助手） |
| 🎯 **核心定位** | 面向 OPC（One Person Company）的全能数字资产与 AI 分身平台 |
| 🏗️ **技术基座** | React 19 + TypeScript + Vite 7 + Supabase + Three.js |
| 📐 **适配终端** | PC 桌面端 + 移动端双端响应式 |
| 🎨 **主题系统** | 5 套主题一键切换（极简 / 赛博 / 艺术 / 童趣 / 复古） |
| 📊 **项目规模** | 9 个页面 · 60+ 组件 · 80+ 源码文件 · 25 个 npm 依赖 |

### 3. 用户画像

| 画像标签 | 描述 | 核心需求 |
|----------|------|----------|
| 🧑‍💻 **独立开发者** | 18-35 岁，有技术能力，需要展示作品 | 在线简历 + 作品集 + 技能树可视化 |
| 📱 **自媒体创作者** | 运营多平台账号，需统一管理 | 新媒体运营面板 + 数据分析 + 内容库 |
| 💼 **自由职业者** | 个人品牌建设需求强烈 | 个人主页 + 数字名片 + 社交矩阵 |
| 🎓 **求职者/学生** | 需要精美简历和知识管理 | 在线简历导出 + 知识库 + AI 学习助手 |
| 💕 **情侣/生活记录者** | 珍藏生活记忆 | 恋爱时间轴 + 旅行地图 + 时光相册 |

### 4. 痛点与价值

| 😣 用户痛点 | 🔧 传统方案 | ✅ Opclaw 解法 | 💡 价值亮点 |
|------------|------------|---------------|------------|
| IP 展示分散 | 简历 Word + GitHub + 各社交平台 | 一站式主页 + 简历 + 作品集 + 社交矩阵 | 个人品牌一体化 |
| 知识碎片化 | Notion / 语雀 / 本地文件分散 | 知识库三栏布局 + 富文本 + RAG 检索 | AI-native 知识管理 |
| AI 不个性化 | 通用 ChatGPT 不懂用户知识 | 自研 RAG + 声音克隆 + 3D 数字分身 | 懂你的 AI 分身 |
| 生活记忆无归处 | 截图 + 相册 + 零散记录 | 恋爱时间轴 + 中国地图轨迹 + 相册 | 情感数字化珍藏 |
| 工作工具割裂 | 电商/新媒体/工具各用不同 SaaS | 电商 + 新媒体 + 百宝箱一体化 | 效率工具聚合 |

### 5. 竞品调研

| 竞品 | 定位 | 优势 | 劣势 | Opclaw 差异化 |
|------|------|------|------|--------------|
| 🔗 **Linktree** | 社交链接聚合 | 简洁、快速搭建 | 功能浅，无知识管理/AI | 功能深度远超，五合一平台 |

| 🌐 **About.me** | 个人主页 | 简洁美观 | 功能单一，无扩展 | 九大功能模块全覆盖 |
| 📊 **飞书多维表格** | 团队协作/数据管理 | 企业级协作 | 非个人向，学习成本高 | 零代码、面向超级个体 |

## 🏗️ 二、技术架构

### 前端技术栈

| 技术层 | 技术选型 | 版本 | 作用 |
|--------|---------|------|------|
| ⚛️ UI 框架 | React | 19.2.0 | 函数组件 + Hooks 响应式 UI |
| 📘 类型系统 | TypeScript | 5.9.3 | 静态类型检查，代码可维护性 |
| ⚡ 构建工具 | Vite | 7.3.1 | ESM + HMR 热更新极速开发 |
| 🎨 样式方案 | Tailwind CSS | 4.1.18 | 原子化 CSS，v4 Vite 插件模式 |
| 🎭 动画引擎 | Framer Motion | 12.34.1 | 页面过渡 / 微交互 / 布局动画 |
| 🧭 路由管理 | React Router | 7.13.0 | Hash 路由 + 页面跳转 |
| 🖼️ 图标库 | Lucide React | 0.574.0 | 1000+ SVG 图标 |

### 后端 & 部署

| 技术 | 用途 |
|------|------|
| ☁️ **Supabase** | BaaS（PostgreSQL + Auth + Storage + RPC + RLS 行级安全） |
| 🚀 **Vercel** | 免费静态站点部署 + CDN 加速 |
| 🗃️ **LocalStorage** | 客户端数据缓存 + 乐观预渲染 |

### AI & 3D 技术

| 技术 | 用途 |
|------|------|
| 🧠 **RAG Engine（自研）** | 关键词提取 + TF-IDF 相似度 + 上下文检索 + 意图识别 |
| 🎮 **Three.js + R3F** | WebGL 3D 角色渲染（眨眼/呼吸/口型动画） |
| 🗣️ **Web Speech API** | 语音识别 + 语音合成，多模态交互 |
| 📊 **ECharts** | 折线 / 柱状 / 饼图 / 雷达图数据可视化 |
| ✏️ **Tiptap** | 富文本编辑器（图片/链接/占位符） |
| 🃏 **html2canvas** | DOM 截图，数字名片图片生成 |

### 架构分层

```
┌─────────────────────────────────────────────────┐
│                   展示层 (View)                  │
│   9 Pages + 60+ Components + Framer Motion      │
├─────────────────────────────────────────────────┤
│                   状态层 (State)                  │
│   ThemeContext + AuthContext + SettingsContext    │
├─────────────────────────────────────────────────┤
│                   服务层 (Service)                │
│   RAG Engine + Supabase Client + Storage API    │
├─────────────────────────────────────────────────┤
│                   数据层 (Data)                   │
│   Supabase PostgreSQL + Mock Data + LocalStorage│
└─────────────────────────────────────────────────┘
```
