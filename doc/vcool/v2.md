2. 过程中有没有遇到什么特别不顺、卡住很久的情况？后来是怎么破局的？
- **“看得见点不到”的编辑器层级风暴：** 
  - 现象：将页面配置为“可直接编辑态”时，因最外层限制 `overflow: hidden` 并拥有繁杂的动画属性，导致下面包裹的所有保存、取消按钮完全失效，深陷层级堆叠的泥潭。
  - 破局：彻底废弃嵌套 CSS 去改层级的方向。通过 AI 一针见血地引入 **React Portals** 方式，把编辑交互层强行飞离原组件结构，顶级挂载到 `body` 上。完美绕开了所有限制，实现了全栈任意处的直接编辑。
- **打印页面（PDF预览版）一团糟：**
  - 现象：在 UI 端光鲜亮丽，但遇到快捷键打印生成简历页面却发生大量布局被截断、冗余的导航空白。
  - 破局：使用专门的 `@media print` 修改打印行为环境：用 CSS 强行藏起了动效、导航等无用噪音元素；给对应大块增加 `page-break-inside: avoid` 与解除高度占位符限制（改为 `height: auto`），输出了一份毫无瑕疵的专业 PDF。

3. 有什么你觉得这次写得特别顺，或者下次还能直接复用的东西？
- **Portal 弹窗装饰器（EditableWrapper 模式）**：真正解耦的利器。只需要往各种展示框外部套用一层，无需触碰内部代码，就能提供就地高质感编辑功能，这个结构以后可以在任何项目中做降维打击复用！
- **极度有效的设计系统 Prompt**：“请输出高质量且具有现代级 Apple 审美水准的界面。必须使用 Vanilla CSS。不可缺失：圆润玻璃拟物态(backdrop-filter)、毛玻璃的高光流转、微小的 hover 回弹交互，并丢弃一切直男式的原始蓝红色彩搭配。”套上这个词，UI 的高级感立竿见影，无需过多返工调教。

四、本周随手记（选填）
- **关于重构的心得**：遇到 CSS 排版里牵扯太多 z-index 或被 overflow 卡的死死的时候，不要死磕，直接选择 Portal 将视图“脱轨”提升维度，比找几层父元素去删要快得多。
- **AI 代码精准化法则**：让大模型（助手）开始写长篇逻辑或补全代码前，先把 `types.ts` 等类型约定单独喂给它。本周经验发现一旦 AI 掌握了 Type 或者 Schema，再让它去写数据或方法，几乎没有空指针或字段不存在的智障报错，生产效率恐怖提升。
- **好用的小改动**：把用户卡片上原本傻楞干瘪的 ID 砍掉，取而代之加入了计算年龄与性别的胶囊式亮眼标签（Tag），非常讨巧的设计；而且把原本死板的“游客访问”通道变成了一条具有彩虹极简流转光圈的边框后，首页的高级感觉立马起飞了！
