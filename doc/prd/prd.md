# 🌈 Opclaw 产品需求文档 (PRD)


## 一、📋 产品概述

### 1. 📝 项目背景

随着AI技术爆发与个体经济崛起，**"OPC超级个体"（One Person Company）** 成为新趋势——一个人即可完成内容创作、产品开发、社群运营等工作。然而，超级个体面临的核心矛盾是：

- **数字资产碎片化**：知识笔记散落各平台、作品集无统一展示、社交账号无法聚合
- **个人IP缺阵地**：简历、作品、技能树分散，难以一站式展现专业形象
- **AI能力未集成**：大模型能力与个人知识库割裂，无法形成"懂我"的AI助手
- **生活记录无归处**：恋爱纪念、旅行轨迹、朋友圈动态缺少统一记录空间
- **工具切换成本高**：学习、工作、生活工具分散，频繁切换降低效率

市场上缺乏一款**面向超级个体、融合个人展示+知识管理+AI赋能+生活记录**的一站式数字宇宙平台。

### 2. ✨项目简介

**Opclaw** 是一款基于 React 19 构建的现代化全栈 Web 应用（PC/移动端），融合个人主页（简历）、学习空间（知识库）、工作助手、生活记录、AI分身于一体，支持5种主题一键切换、流畅动画与高性能渲染，让每个人快速拥有高颜值、可扩展的数字宇宙。

**一句话定位：** 面向超级个体的全能数字资产与AI分身助手——一个人的数字宇宙，从这里开始。

**核心价值主张：**

-  **个人IP一站式展示**：简历+技能树+作品集+社交矩阵，一张名片打天下
- **知识资产沉淀管理**：知识库+富文本编辑+RAG检索，让学习有迹可循
-  **AI分身能力赋能**：声音克隆+形象复刻+3D角色+智能对话，数字分身替你工作
-  **生活情感数字珍藏**：恋爱记录+旅行地图+时光相册，记忆永不褪色
-  **工作效率加速器**：新媒体运营+电商管理+百宝箱，提效工具一站集成

| **界面风格单调** | 大多数产品只有亮/暗两套主题 | 5套精心设计主题（极简/赛博/艺术/童趣/复古） | 个性化表达维度5x |

### 5. 🔍 竞品调研

| 竞品 | 定位 | 优势 | 不足 | 与Opclaw差异 |
|------|------|------|------|--------------|
| **Notion** | 全能知识管理 | 生态完善、模板丰富 | 无AI分身、无个人主页、无生活记录 | Opclaw有AI分身+个人展示+生活模块 |
| **Linktree** | 个人链接聚合 | 简单易用、社交入口 | 功能单一、无知识库、无主题系统 | Opclaw功能深度远超链接聚合 |
| **About.me** | 个人主页 | 快速建站、名片风格 | 无知识管理、无AI、无生活记录 | Opclaw是全功能平台而非单页 |
| **Character.AI** | AI角色对话 | 角色丰富、社区活跃 | 无个人知识库、无资产管理 | Opclaw的AI分身基于个人知识，更懂你 |
| **飞书文档** | 协作知识管理 | 企业级协作、生态整合 | 偏B端、无个人IP展示、无生活记录 | Opclaw面向C端个体，更聚焦 |
| **GitHub Pages** | 开发者主页 | 开发者生态、代码驱动 | 技术门槛高、功能单一 | Opclaw零代码、可视化、多模块集成 |

**竞品格局总结：** 市场上现有产品要么只做知识管理（Notion），要么只做个人展示（Linktree/About.me），要么只做AI对话（Character.AI），**没有一款产品将个人IP展示+知识管理+AI分身+生活记录+工作助手融合于一体**。Opclaw正是填补这一空白。

### 🤖 2.自研RAG知识检索引擎

> 基于关键词检索+本地知识库的智能问答引擎

**核心API：**

```typescript
// 检索相关内容
ragEngine.search(query: string, topK?: number)
→ { relevantContent: string[], sourceModules: string[], confidence: number }

// 生成智能回复（含推荐/方法/介绍等多种回复模式）
ragEngine.generateResponse(query: string) → string
```

**特色：**
- 🧠 关键词匹配+语义理解双引擎
- 📚 检索范围覆盖学习、生活、娱乐全模块知识
- 💡 多种回复模式（推荐/方法/介绍/闲聊）
- 🔒 本地运行，无需外部API调用

### 🔐 5 .Supabase云端认证系统

> 邮箱+用户名双模式登录+RLS行级安全

**认证流程：**
```
用户访问 → AuthContext初始化 → LocalStorage乐观预渲染
  → supabase.auth.getUser()验证
  → 已登录 → 同步profiles表资料
  → 未登录 → 100ms后弹出AuthModal

用户名登录 → RPC查询邮箱 → signInWithPassword → 登录成功
用户注册 → 检查用户名唯一 → signUp → 创建profiles记录 → 自动登录
```

**数据库设计：**
```sql
profiles (
  user_id UUID PRIMARY KEY,
  username TEXT UNIQUE,
  email TEXT,
  avatar_url TEXT,
  background_url TEXT,
  bio TEXT,
  created_at TIMESTAMP,
  updated_at TIMESTAMP
)
```

**安全策略：** Row Level Security (RLS) 行级安全策略，用户只能访问和修改自己的数据。
